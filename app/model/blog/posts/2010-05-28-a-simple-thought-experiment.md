title: "A Simple Thought Experiment"
---

[rentzsch](http://rentzsch.tumblr.com/post/641320178/a-simple-thought-experiment):

> Fifteen years from now, will you be using the same language(s) as you’re using today?
> 
> If not, why not?

This is interesting. Personally, my answer is "Hopefully not!"

ObjC is great for its speed and ability to interface with C, but that's where the funs stops. Sure, compared to Java and C++ ObjC is quite a refreshing change and cures a lot of headaches. It's simplicity is admirable. But it's verbosity is intolerable, in my opinion.

Python is amazingly fun to write in, it's so much prettier to read and a lot more concise, but it's got its own oddities too (I've heard the GIL limits the threading capabilities of Python). Also, Python doesn't treat me like an irresponsible child. I love its motto of "we're all consenting adults, here."

I hope in 15 years, I'll be writing in a language as beautiful and elegant and dynamic and powerful as Python, but with the speed and power and low-level control as Objective-C. (And I want to use it to write full-fledged Cocoa apps.)
