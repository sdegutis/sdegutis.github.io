Clojure for web apps, Part 2 This is Part 2 in the series that explains why I believe Clojure is the best language for writing web apps. Also see [Part 1](/blog/2013-10-22-clojure-for-web-apps-p1) and [Part 3](/blog/2013-12-23-clojure-for-web-apps-p3). ### It's perfect for templating Consider HTML templates. In Ruby, you'd use Haml or Erb. But in Clojure, you can just use Clojure. Take this small web page for example: ```clojure (html [:div#title [:h2 "Hello World"] [:p "Welcome to 1995."] [:p "We have pretty okay music. " [:i "And spicy food."]] [:a {:href "mailto:nobody@ever.srsly"} "Contact webmaster."]]) ``` This is pure Clojure code. There's only one function call, `html`, which is provided by [Hiccup](https://github.com/weavejester/hiccup)to translate some Clojure data into an HTML string. (There's a similar library to translate Clojure to CSS, called [Garden](https://github.com/noprompt/garden).) This isn't just about the convenience of not having to write a parser. This means you're always in the same language. If you wanted to "render a partial template" in Hiccup, you just call a function inside your Hiccup code that returns more Hiccup code. It's just a Clojure function after all. You can even pass arguments to it. If you wanted to loop to transform a data structure into some Hiccup code for each element, you just use Clojure's `for`. This makes Clojure perfect for templating.
