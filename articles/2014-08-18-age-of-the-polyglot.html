Age of the Polyglot For years I've been trying to find The One True Programming Langauge that I could use for every program, for any task I'll ever have. But these days, I'm pretty sure this language just doesn't exist. I've tried to write OS X apps using Lisp. I've tried to write iOS apps using Ruby. I've tried to write Windows apps using Clojure. I've tried to write web apps in Objective-C. Sure, these are all possible. But to me it just never felt natural, or right. In each instance, I felt like the tools were fighting me rather than working for me. Instead of just wrestling the platform, I was also having to fight the Frankenstein bridge, the bridge that was shoehorned in and never quite meant to be in the first place. So from now on, when writing an OS X app, I'll use Objective-C. For a Windows app, C#. For an iOS app, nothing, because I don't write iOS apps (unless the money is good enough). For web apps, I'll use Rails and thus Ruby, and for the front-end I'll use plain old JavaScript. For short one-off command line scripts, I'll try to use Bash if I can. If I need more advanced features than Bash offers, I'll probably use Python or Ruby. Or it may be a job for C, in which case I might even need to write a little Makefile for it. I might also need to embed an extension language into it, in which case I'll use Lua. Over the years I've done all these. And yes, I ended up learning more than I set out to learn in the first place. But each time, the result felt good, and pure, and simple, and natural. It just felt _right_.
