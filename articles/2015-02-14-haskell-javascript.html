Ways to compile Haskell to JavaScript This'll be shorter than the last thing, because I think I don't like Haskell so I don't want to put too much stock into this kind of research. But I'm writing it in case I ever forget why I don't like it: ### PureScript PureScript has eager evaluation, JS-like records, easy(ish) JavaScript FFI, type classes, and an Eff monad in place of IO. Has decent number of libraries. ### Elm Elm is like PureScript but comes with a special runtime, more difficult FFI, and is geared mostly towards being a React.js replacement. Also, Elm had some really neat Canvas-based drawing libs, and was pretty easy to install and use, but seems very isolated. Plus it has elm-html which looks like it'll probably deprecate the Canvas-based GUI lib soon. ### GHCJS GHCJS is actually real Haskell, and seems to still be maintained and active.
