Some thoughts on GUIs I spent a few years learning Visual Basic, then a few years learning Xcode and Cocoa. But lately the web is taking over as the easiest GUI to style and the most cross-platform GUI toolkit. For a while I resisted it on principle, because HTML and CSS and JS have so many flaws. But I tried node-webkit and it was pretty cool. It lets me use Node's JS API to do things like read/write to the file system. Then I heard about Atom Shell and it's even cooler. So, my new favorite GUI toolkit consists of: 1. [Atom Shell](https://github.com/atom/atom-shell) for GUI 2. [Mithril](http://lhorie.github.io/mithril/) for dynamic HTML generation 3. [Stylus](http://learnboost.github.io/stylus/) [(compiled in the browser)](http://learnboost.github.io/stylus/docs/js.html) for CSS generation 4. [Immutable](http://facebook.github.io/immutable-js/) and [UUID](https://www.npmjs.com/package/uuid) for an ad-hoc JSON database 5. [Normalize](http://necolas.github.io/normalize.css/) just to give consistent defaults 6. JavaScript ### Nothing needs to be compiled! The only tricky thing I do here is compile Stylus dynamically in the browser. Everything else is pure JS. ### Why JavaScript? Mostly just to simply the build process. I tried LiveScript which is mostly just syntactic sugar but also adds some cool FP things like auto currying, but at the cost of being a little too syntactically sweet. I tried ClojureScript but the only benefit I can see here is macros (which I always avoid) and negligibly more consistent syntax. Immutable.js covers my data needs well enough. Plus, accessing JS is super awkward in ClojureScript. So far, once you learn how to work around JavaScript's caveats (every language has caveats, don't fool yourself), JS's main flaw is how verbose it is, especially with function syntax. But ES6 is addressing that. ### Do I really need Normalize? Probably not. Atom Shell uses Chromium. And if that ever changes, I can restyle what needs restyling, which won't be much. Normalize does give me some sane defaults, like button font size. So, slightly less styling for me to do. But not enough to matter, so I'm considering ditching it anyway. ### Not automatically cross platform Even though Atom Shell works anywhere, there's a lot of genuine hard work still involved in making an app cross platform. But I don't need that feature right now. ### Open questions 1. Is Immutable + UUID a sufficient replacement for a Mongo-like database? Time will tell.
