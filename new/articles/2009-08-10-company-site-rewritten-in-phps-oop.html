Company site rewritten in PHP's OOP... On August 1st, I went live with [thoughtfultree.com](http://www.thoughtfultree.com), which had all my updated apps (and some new ones), and now uses PayPal as my store instead of eSellerate. But, even though my software was rewritten, the site was still the same old minimal, functional site I coded up almost a year ago. (It sucked big time). This is all fine and dandy, until I realized I was copy/pasting so much of my code due to not taking the site seriously, that it was becoming very very difficult when I started to code my own licensing stuff (for PayPal's IPN). So, I had decided to learn PHP's Object-Oriented stuff in PHP 5.x, which proved to be amazingly fun experience. Now the entire site is written using proper MVC paradigm, including a minimal "template engine" I wrote specifically for my own use, in the form of a class called View. The site is just so much cleaner now, on the back-end, and I'm really pleased with how easy it's been to make both minor and major changes here and there, now that it's all done The Right Way ™. One thing that really bothered me, though, was 5.2.9's lack of late static binding. This specifically means that a lot of patterns I'm used to in Objective-C, such as an abstract class that doesn't know about its subclasses, are so much more difficult and require a lot more redundancy in the form of variables and parameters, not to mention lack in efficiency. Let me 'splain... Let's say I have a class A and the classes B, C, and D which all inherit from A. Now, I'll want to make `static function factory()` inside A, which would be accessed via subclasses only, in the form `B::factory()` or `D::factory()`. Unfortunately, I could not use the keyword `self` inside this function and get B or D, as it would always return A, since it's literally static and whatever class it's inside of will always be the value of `self`. This is much different than in Objective-C, where we can code up a class method `+(id)factory`, call it from/on subclasses, and have the value of `self` inside it be the actual subclass, such as `[D factory]` where `self` will be `D`, not `A`. This seems like a pointless little detail until, but what it means is that we cannot treat PHP's static class functions as if they were Objective-C's class methods. They act completely differently. (A similarly annoying limitation exists for static class variables.) So I've worked around it for the time being using a little ugliness, but besides that, the site is so clean I could eat off of it. And that makes me happy. So now I'll be going back to updating my apps and putting them online!
