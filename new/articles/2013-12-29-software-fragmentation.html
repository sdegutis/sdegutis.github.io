Software Fragmentation I love how free we are to make software at basically any level of the stack. I could make my own OS, my own language, my own web browser or text editor. But that also means so can everyone else. And since none of us can agree on anything, that means we have tons of incompatible, fragmented software. It's really frustrating that we're duplicating all our efforts in multiple places just to write the same basic things. Ultimately I think this is a good thing. The fact is, we're not going to agree on everything, mostly due to varying levels of experience. Sure, I could write a programming language today, but I would trust the authors of Clojure and C# to make better design decisions than I can. Besides, imagine a world where a single OS is mandated, and a single language. There would be no room for innovation! What if we realize major flaws in that language? What if we discover major design flaws in the OS? I love that we have a free market in software right now, because it means survival of the fittest software. I would switch to Microsoft's OS in a heartbeat if it were better. This quote from [someone on Hacker News](https://news.ycombinator.com/item?id=6975049) sums up how I feel (except for the Haskell and Rust part): > As a Linux user myself, the main reason why I switched from Windows is that all the exciting advances in programming languages are happening in \* nix-land, mainly Haskell and Rust. If there were some super-duper-awesome language for which Windows were the first-class platform, I would seriously consider switching back. I do not give a single damn about software freedom, technical excellence trumps all else. So yeah, we have tons of fragmentation. Tons of duplicated efforts. But it's probably for the better. In 20 years, a clear winner will probably have emerged, and we'll probably all be writing for that OS in that language with that standard library. And the free software market is probably the only way to get there.
